# Blog Autopilot — Design Document

**Date:** 2026-03-05
**Status:** Approved

## Overview

Autonomous blog content generation system that creates, translates, and publishes SEO-optimized blog posts using Claude Sonnet 4.6. Integrates into the existing Express server as a cron job and extends the content-manager MCP server with autopilot tools.

## Architecture

```
Express Server
├── node-cron (L/M/V 9am) ──► Blog Autopilot Service
│                               1. Check season (feb-sep)
│                               2. Check last run (DB)
│                               3. Pick topic
│                                  ├─ WhatsApp chatbot data (real customer questions)
│                                  ├─ Feedback loop (page_visits analysis)
│                                  └─ Claude knowledge (niche expertise)
│                               4. Generate article (Claude Sonnet 4.6)
│                                  ├─ With knowledge base context
│                                  └─ With existing boats/destinations/posts for internal links
│                               5. Auto-review (Claude self-critique + SEO audit)
│                               6. Translate to 8 languages
│                               7. Fetch image (Unsplash)
│                               8. Insert as draft in DB
│                               9. Log execution
│
├── node-cron (hourly) ──► Auto-publish mature drafts (after delay)
│
└── content-server.ts MCP ──► 7 existing + 7 new autopilot tools
```

## Topic Selection Engine (Priority Order)

1. **WhatsApp chatbot data** — Analyze `chatbot_conversations` and `ai_chat_sessions` for frequently asked questions. Real customer demand = best content topics.
2. **Feedback loop** — Query `page_visits` to identify high-performing content categories/clusters. Prioritize more of what works.
3. **Claude niche knowledge** — Fill gaps in the topical cluster strategy using Claude's knowledge of "boat rental Costa Brava" niche.

Validation: Claude analyzes existing `blog_posts` titles/slugs to avoid duplicate topics.

## Content Generation Pipeline

```
1. Select topic (WhatsApp + visits + Claude)
2. Generate article in Spanish (with knowledge base + boats/destinations context)
3. Self-review pass (Claude critiques and improves)
4. SEO audit (reuses existing get_seo_audit logic)
5. If seoScore < 70 → regenerate (max 2 retries)
6. Translate to 7 other languages
7. Fetch Unsplash image
8. Save as draft with publishDelayHours countdown
9. Auto-publish when delay expires
```

## Multi-language Strategy

Store translations as JSONB fields on the same blog_posts record:
- `titleByLang`: `{ es: "...", en: "...", fr: "...", ... }`
- `contentByLang`: `{ es: "...", en: "...", ... }`
- `excerptByLang`: `{ es: "...", en: "...", ... }`
- `metaDescByLang`: `{ es: "...", en: "...", ... }`

Original fields (`title`, `content`, etc.) remain as Spanish defaults for backward compatibility. Frontend uses `*ByLang[language]` with fallback to original fields.

**Supported languages:** es, en, fr, de, it, nl, ru, ca

## Topical Clusters

Clusters group related posts (1 pillar + 5-8 satellites) with internal linking:
- System auto-creates clusters based on niche categories
- Generates pillar post first, then satellites
- Each satellite links to pillar and sibling satellites
- When cluster completes, creates a new one
- Traffic analysis can trigger additional satellites for high-performing clusters

## Content Refresh

- 1 in every N runs (configurable, default: 4) targets existing posts instead of new ones
- Selects posts >30 days old with low traffic (from page_visits)
- Rewrites content, improves SEO, updates `updatedAt`
- Logs as type "refresh" in autopilot_log

## Seasonality

- **Active season:** February through September (configurable)
- Feb-March = pre-season content (time for Google to index before tourist season)
- April-September = peak season
- October-January = off-season (no auto-generation)
- `autopilot_generate_now` overrides season check for manual generation

## Database Schema Changes

### New fields on `blog_posts`

| Field | Type | Description |
|-------|------|-------------|
| titleByLang | jsonb | Translations `{ es, en, fr, de, it, nl, ru, ca }` |
| contentByLang | jsonb | Translated content |
| excerptByLang | jsonb | Translated excerpts |
| metaDescByLang | jsonb | Translated meta descriptions |
| clusterId | varchar | FK to blog_clusters |
| isAutoGenerated | boolean | Created by autopilot (default: false) |
| seoScore | integer | Auto-audit score (0-100) |

### New table: `blog_clusters`

| Field | Type | Description |
|-------|------|-------------|
| id | varchar (uuid) | PK |
| name | varchar(255) | Cluster name |
| pillarPostId | varchar | FK to blog_posts (pillar article) |
| keywords | text[] | Primary keywords |
| plannedTopics | jsonb | Satellite topics planned |
| completedCount | integer | Satellites completed |
| isComplete | boolean | All satellites written |
| createdAt | timestamp | Created date |
| updatedAt | timestamp | Last update |

### New table: `blog_autopilot_config`

| Field | Type | Default |
|-------|------|---------|
| id | varchar (uuid) | PK |
| isEnabled | boolean | true |
| cronSchedule | varchar | `0 9 * * 1,3,5` |
| model | varchar | `claude-sonnet-4-6` |
| languages | text[] | 8 languages |
| maxPostsPerWeek | integer | 3 |
| seasonStartMonth | integer | 2 (February) |
| seasonEndMonth | integer | 9 (September) |
| publishDelayHours | integer | 24 |
| minSeoScore | integer | 70 |
| refreshRatio | integer | 4 |
| unsplashEnabled | boolean | true |
| useWhatsappTopics | boolean | true |
| createdAt | timestamp | now() |
| updatedAt | timestamp | now() |

### New table: `blog_autopilot_log`

| Field | Type | Description |
|-------|------|-------------|
| id | varchar (uuid) | PK |
| postId | varchar | FK to blog_posts (null if failed) |
| type | varchar | `new` / `refresh` / `manual` |
| topicChosen | text | Selected topic |
| clusterName | varchar | Cluster context |
| keywordsUsed | text[] | Keywords targeted |
| modelUsed | varchar | Claude model used |
| tokensInput | integer | Input tokens consumed |
| tokensOutput | integer | Output tokens consumed |
| seoScore | integer | Audit score achieved |
| status | varchar | `success` / `error` / `regenerated` |
| errorMessage | text | Error details (if failed) |
| createdAt | timestamp | Execution time |

### New table: `blog_autopilot_queue`

| Field | Type | Description |
|-------|------|-------------|
| id | varchar (uuid) | PK |
| clusterId | varchar | FK to blog_clusters |
| topic | text | Proposed topic |
| keywords | text[] | Target keywords |
| type | varchar | `pillar` / `satellite` / `refresh` |
| priority | integer | Execution order |
| status | varchar | `planned` / `generating` / `generated` / `discarded` |
| scheduledFor | timestamp | Planned date |
| postId | varchar | FK to generated post |
| createdAt | timestamp | Created date |

## MCP Tools (added to content-server.ts)

| Tool | Description |
|------|-------------|
| `autopilot_status` | Current state, season, next run, monthly stats, costs |
| `autopilot_configure` | Update schedule, season, model, delay, enable/disable |
| `autopilot_generate_now` | Force immediate generation (ignores season) |
| `autopilot_queue` | View/manage planned topics (reorder, discard) |
| `autopilot_history` | Execution log with costs and performance |
| `autopilot_refresh_post` | Force rewrite of a specific low-performing post |
| `autopilot_cluster_status` | Active clusters, progress, traffic per cluster |

## Cost Estimate

At 3 posts/week (12/month), 8 languages each:
- ~96 Claude Sonnet calls/month (12 articles × 8 languages)
- Input: ~3,000 tokens × 96 = 288K tokens → ~$0.86
- Output: ~4,000 tokens × 96 = 384K tokens → ~$5.76
- **Total: ~$6.62/month**

Unsplash: Free tier (50 requests/hour, we use ~12/month)
